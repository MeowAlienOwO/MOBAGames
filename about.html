GRP - MOBA GAME
Project Introduction
A MOBA (Multiplayer Online Battle Arena) game is a real-time strategy game involving two teams of five players fighting over control of a map. This map contains multiple enemy towers and a base that must be destroyed, and groups of AI-controlled non-player characters that fight for each team. MOBA games currently feature heavily in eSports with the two most popular titles being DotA2 and League of Legends, both with millions of players.

MOBA games have many interesting design choices. How games play out is also interesting from a data-mining perspective. Unfortunately, all major MOBA games are closed source with both the code and the data the games generate being closely guarded secrets. There is currently no significant open source MOBA. This project is to write the fundamentals of a working MOBA game which can used as a basis for research and future student projects.

We will conduct this project in Java using multi-threading and networking techniques. This project will focus on creating the bare minimum basic experience that works. In particular:
  • Graphics will be limited to a simple top-down 2D view of colored icons.
  • No physics or advanced special effects.
  • No sound.
  • Most of the game mechanics can be the same as are commonly used in other MOBA games (game mechanics aren't copyrightable but the actual names, graphics, etc are).
  • Some of the more tricky network issues can be glossed over (e.g, disconnections and re-connections).
